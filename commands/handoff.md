# Handoff — Generate Context Package

Create a handoff document that gives another person or AI session everything they need to continue this project without losing context.

## Step 0: Validate sitrep location (MANDATORY)
1. Verify `sitrep/PROJECT_PLAN.md` and `sitrep/STATUS_REPORT.md` exist
2. If missing → search, move, or report. Do NOT proceed until both are in `sitrep/`

---

## Step 1: Archive previous handoff
1. Check if `sitrep/HANDOFF.md` already exists
2. If it does:
   - Create `sitrep/history/handoffs/` directory if it doesn't exist
   - Read the old HANDOFF.md to find the session number from its header
   - Move it to `sitrep/history/handoffs/handoff-session-[N].md`
   - Print: "📦 Archived previous handoff → history/handoffs/handoff-session-[N].md"
3. If it doesn't exist, skip this step

---

## Step 2: Read everything
- Read `sitrep/MANIFEST.md` (if exists — for version context)
- Read `sitrep/PROJECT_PLAN.md`
- Read `sitrep/STATUS_REPORT.md`
- Read `CLAUDE.md` (if it exists)
- Scan the repo structure (top 2 levels)
- Check recent git log (last 10 commits)

Extract the project name from PROJECT_PLAN.md. Use as [PROJECT].

---

## Step 3: Generate handoff document

Create `sitrep/HANDOFF.md` with this structure:

```markdown
# [PROJECT] — Handoff

> **Generated:** [date]
> **Last session:** [session number from status report]
> **Status:** [current phase] — [X/total tasks — Y%]
> **sitrep version:** [from MANIFEST.md or "unknown"]

---

## What This Project Is
[2-3 sentences from PROJECT_PLAN.md product vision. Plain English, no jargon.]

## Tech Stack
[List from PROJECT_PLAN.md or inferred from repo]

## Current State
[What's working right now. What can you run/see/test today.]

## What Was Just Done (Last Session)
[From the most recent session log entry in STATUS_REPORT.md]

## What's Next
[The queued tasks from the active sprint. Be specific — task IDs and descriptions.]

## Active Decisions & Context
[Key decisions from PROJECT_PLAN.md that affect upcoming work. Things the next person MUST know to avoid re-debating or breaking.]

## Watch Out For
[Known gotchas, fragile areas, things that are half-done, env quirks, anything that could trip someone up.]

## Architecture Quick Reference
[Folder structure, key files, how to run the project, how to run tests. Keep it terse.]

## Blockers
[Any ❌ items from STATUS_REPORT.md, or "None"]

---

_Generated by `/handoff`. Full details in `sitrep/PROJECT_PLAN.md` and `sitrep/STATUS_REPORT.md`._
```

---

## Step 4: Tailor to audience

If $ARGUMENTS contains "human":
- Write for a human developer joining the project
- Include setup instructions (how to clone, install, run)
- Include "questions you should ask before starting"

If $ARGUMENTS contains "ai":
- Write for an AI assistant in a new session (Claude.ai, new Claude Code session, etc.)
- Include exact file paths to read first
- Include "start by reading these files in this order: ..."
- Be more structured, less narrative

If no arguments:
- Default to "ai" format (most common use case)

---

## Step 5: Print and commit

Print:
```
=== [PROJECT] HANDOFF READY ===
File: sitrep/HANDOFF.md
Target: [human/ai]
Project: [X/total tasks — Y%]
Last session: [N] — [summary]
Next up: [queued tasks]
Archived: [previous handoff path or "none"]

Share this file to onboard anyone into the project.
=================================
```

Commit:
```bash
git add sitrep/
git commit -m "sitrep: handoff — session [N]"
```
